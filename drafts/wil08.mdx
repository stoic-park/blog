어쩌다 8주차가..?

휴강 주를 지나고 어느새 8주차가 됐습니다

개인사로 인해 아주 힘든 8주차를 보내고 3번째 챕터인 테스트코드 챕터를 마무리하며 오랜만의 회고 시간을 가져보겠습니다

## 1. 이번 챕터 돌아보기

2주간의 학습에서는 프론트엔드 테스트의 전반적인 흐름을 이해하고, 다양한 테스트 기법을 실습해보았습니다. 특히 단위 테스트와 통합 테스트, E2E 테스트와 시각적 회귀 테스트 등 프론트엔드 테스트의 종류에 대해 체계적으로 학습하였고, @testing-library/react, vitest, MSW 등을 활용한 실습을 통해 실제 테스트 시나리오를 구성하고 실행하는 경험도 쌓을 수 있었습니다. 또한 테스트 전략 수립과 TDD 기반 개발 방식에 대해서도 고민해보며, 테스트 코드의 중요성과 실무 적용 가능성에 대해 깊이 있게 탐구할 수 있는 시간이었습니다.

특히 팀 활동 시간에는 테스트 전략에 대해 각 팀원들이 실제 회사에서 겪은 사례들을 공유하고, 이를 바탕으로 이번 과제를 어떤 전략으로 진행하면 좋을지 함께 고민해보는 시간을 가졌습니다. 그동안 다소 막연하게 느껴졌던 테스트 코드 작성에 대해 명확한 방향을 잡는 데 큰 도움이 되었고, 실무에서 테스트를 어떻게 적용하고 확장해 나갈 수 있을지에 대한 시야도 넓힐 수 있었습니다.

## 2. 이번 챕터의 목표

이번 챕터의 목표는 프론트엔드 테스트의 기본 개념과 종류를 이해하고, 단위 테스트부터 E2E 테스트까지 다양한 테스트 도구와 기법을 직접 실습해보며, 테스트 전략 수립과 협업 경험을 통해 테스트 코드 작성의 방향성을 잡는 것이었습니다.

## 3. 무엇을..?

7주차에는 유닛 테스트부터 점진적으로 프로젝트의 테스트 코드를 작성해볼 수 있었습니다

프로젝트를 구성하는 각 유틸함수부터 훅까지 단계적으로 테스트 코드를 작성해볼 수 있었습니다.
특히 msw 등으로 데이터를 모킹하는 부분도 경험할 수 있었고, 이부분은 추후에 좀 더 학습해보고 익숙해질 수 있도록 하면 좋을 것 같습니다.

## 4. 아쉬웠던 부분

8주차에 팀원들과 함께 테스트 전략을 수립해보는 시간을 가졌습니다.
다만 이 부분에서 개인적인 일정으로 인해 급하게 과제를 진행하면서(핑계) 애초에 기획했던 전략대로 테스트 코드 작성 플로우를 지키지 못했습니다.

우선 과제의 기본 가이드라인에서도 TDD 작성 플로우를 지키지 못했고, 통합 테스트의 비중을 높게 두고, 단위테스트, E2E 테스트의 비중을 차례대로 할 것을 지키지 못했습니다

과제를 제출할 단계가 되고 보니, 과제 필수 조건에만 집중한 나머지 단위 테스트에 비중이 가장 높게 두고 테스트 코드를 작성하게 되었습니다.

## 5. 마무리

평소 테스트 코드의 중요성에 대해서는 의식적으로 갖고 있으면서도 실제로 작성해보려는 노력은 하지 못했던 것 같습니다.
실무에서도 주로 단위 테스트를 작성하는 수준까지의 경험이 주고, E2E나 시각적 테스트는 경험은 해봤지만, 프로젝트가 진행될수록 개인적인 업무 우선순위에서 밀려 그 끝이 흐지부지 되는 것이 대부분이었습니다

하지만, 이번 챕터를 통해서 테스트 코드에 대한 전반적인 과정을 경험해봄으로써 평소 어색했던 테스트 코드와 조금은 친해지지 않았나 생각해봅니다.

다음 챕터는 `인프라 관점의 성능 최적화` 챕터입니다. 동일하게 평소 중요성을 느끼고 있지만 경험은 부족했던 부분이여서 좀 더 공부해보고 싶은 부분이었습니다.
챕터 잘 마무리해서 좋은 자산이 되기를 기대해 봅니다!
