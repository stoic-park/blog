# 유틸리티 타입에 대해서!

베스트 케이스를 기억해두면 그 어떤 녀석들도 내 타입이야

실무에서 사용하는 유틸리티 타입들에 대해서 정리해봅니다

작은 스펙 → 명시 타입이 최고

반복 패턴 → RequiredBy/OptionalBy + Simplify

PATCH 제약 → AtLeastOne, XOR

드래프트/다단계 → DeepPartial + 제출 시 스키마 검증

외부 응답 → 정규화 후 내부 모델만 사용

id 혼동 방지 → Brand

가독성/리뷰성 → 결과 타입만 export, 복잡한 조건부는 내부에서 처리
