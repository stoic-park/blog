# 이펙티브 타입스크립트 - 댄 밴더캄

## 핵심 메시지

타입스크립트는 단순한 타입 체커가 아니라, 더 나은 코드를 작성하는 도구입니다.

## 주요 인사이트

### 1. 타입 시스템 이해하기

- 타입스크립트는 구조적 타이핑을 사용
- 타입은 값의 집합
- 타입 체커는 컴파일 타임에 오류를 잡아줌

### 2. 타입 추론 활용하기

- 명시적 타입 선언보다 타입 추론 활용
- 타입 추론이 어려운 경우에만 명시적 타입 사용
- `const`와 `as const`의 차이점 이해

### 3. 제네릭과 유니온 타입 활용

- 제네릭으로 재사용 가능한 타입 만들기
- 유니온 타입으로 다양한 케이스 처리
- 조건부 타입으로 동적 타입 생성

## 실무 적용 포인트

1. **점진적 타입 적용**: 기존 JavaScript 코드에 점진적으로 타입 추가
2. **strict 모드 사용**: 엄격한 타입 체크로 더 안전한 코드 작성
3. **타입 가드 활용**: 런타임에 타입 안전성 보장

## 개인적인 생각

타입스크립트를 처음 접할 때는 타입 정의가 번거롭다고 생각했지만, 이 책을 읽고 나서는 타입이 코드의 문서 역할을 한다는 것을 깨달았습니다.

특히 제네릭을 활용한 재사용 가능한 컴포넌트를 만들 때 타입스크립트의 강력함을 실감했습니다. 컴파일 타임에 오류를 잡아주어 런타임 에러를 크게 줄일 수 있었습니다.
