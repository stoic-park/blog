# 클린 코드 - 로버트 C. 마틴

## 핵심 메시지

클린 코드는 단순히 작동하는 코드가 아니라, 읽기 쉽고 이해하기 쉬운 코드입니다.

## 주요 인사이트

### 1. 의미 있는 이름 사용하기

- 변수, 함수, 클래스 이름은 의도를 명확히 표현해야 함
- 검색하기 쉬운 이름 사용
- 한 글자 변수명은 루프 카운터나 반복자에만 사용

### 2. 함수는 작게, 한 가지 일만 하기

- 함수는 20줄 이내로 작성
- 들여쓰기 수준은 2단계 이하
- 함수명은 동사로 시작

### 3. 주석보다는 코드로 의도 표현하기

- 주석은 왜(Why)에 집중
- 무엇(What)은 코드로 표현
- 주석이 필요하다면 코드에 문제가 있을 가능성

## 실무 적용 포인트

1. **리팩토링 습관화**: 작은 변경이라도 리팩토링 기회로 활용
2. **테스트 코드 작성**: 깨끗한 코드는 테스트하기 쉬움
3. **코드 리뷰**: 동료와 함께 코드 품질 개선

## 개인적인 생각

이 책을 읽고 나서 가장 큰 변화는 변수명과 함수명을 더 신중하게 짓게 되었다는 점입니다. 특히 "의도를 드러내는 이름"이라는 개념이 인상적이었습니다.

코드가 읽기 쉬워지면서 유지보수성도 크게 향상되었고, 동료들과의 협업도 더 원활해졌습니다.
